GitHub Upload Guide

E-Commerce Sales Dashboard · Step-by-step from zero to live GitHub Pages URL
1
Install Git (if not installed)
1
Check if Git is already installed
Open your terminal (Command Prompt / PowerShell on Windows, Terminal on Mac) and run:
git --version
If you see something like git version 2.43.0, skip to Phase 2. Otherwise continue below.
2
Install Git
Windows: Download from git-scm.com/download/win → run installer with default settings

Mac: Run in terminal: xcode-select --install

After installing, configure your identity: 
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
2
Create a GitHub Account & Repository
1
Sign up / Log in to GitHub
Go to github.com → Sign up (free) or log in to your existing account.
2
Create a new repository
• Click the green "New" button on your GitHub homepage
• Repository name: ecommerce-dashboard
• Description: Interactive e-commerce sales dashboard — Python, Chart.js
• Set to Public (required for free GitHub Pages)
• ✅ Check "Add a README file"
• Click "Create repository"
3
Organize Your Project Files
1
Create the project folder on your computer
Create a folder called ecommerce-dashboard on your Desktop or Documents. Your folder should look like this:
ecommerce-dashboard/
  ├── index.html     ← The dashboard (downloaded from Claude)
  ├── data_prep.py   ← Python script (downloaded from Claude)
  ├── README.md     ← README (downloaded from Claude)
  └── processed/
      ├── monthly_trend.csv
      ├── category_performance.csv
      ├── regional_breakdown.csv
      └── dashboard_data.json
💡 The processed/ folder is generated when you run python data_prep.py locally. You can also just upload the 3 files (index.html, data_prep.py, README.md) and that's enough for a great GitHub portfolio.
4
Push Files to GitHub
1
Open terminal and navigate to your project folder
# Mac/Linux
cd ~/Desktop/ecommerce-dashboard

# Windows (PowerShell)
cd C:\Users\YourName\Desktop\ecommerce-dashboard
2
Initialize git and connect to GitHub
# Initialize a new git repository
git init

# Add the GitHub repo as remote origin
# (Replace YOUR-USERNAME with your actual GitHub username)
git remote add origin https://github.com/YOUR-USERNAME/ecommerce-dashboard.git
3
Stage all files and make your first commit
# Stage everything
git add .

# Commit with a message
git commit -m "Add e-commerce sales dashboard - Python, Chart.js, Apr 2026"
4
Push to GitHub
# Push to the main branch
git branch -M main
git push -u origin main
GitHub will prompt for your username and password. For the password, use a Personal Access Token (not your account password) — see next step.
5
Create a Personal Access Token (if prompted)
GitHub no longer accepts passwords for git push. To create a token:

1. GitHub → Settings (top-right avatar menu)
2. Left sidebar → Developer settings
3. Personal access tokens → Tokens (classic)
4. Click "Generate new token (classic)"
5. Give it a name, set expiry, check repo scope
6. Click Generate token → copy it immediately (shown only once)
7. Use this token as your password when git asks
⚠️ Store the token somewhere safe (like a password manager). You won't be able to see it again on GitHub.
5
Enable GitHub Pages (Live URL)
1
Go to your repo's Settings
On GitHub, open your ecommerce-dashboard repo → click the Settings tab at the top.
2
Enable Pages
• Left sidebar → Pages
• Under "Branch", select main and folder / (root)
• Click Save
3
Wait ~2 minutes, then visit your live URL
Your dashboard is now live at:
https://YOUR-USERNAME.github.io/ecommerce-dashboard
✅ This is the link to put on your resume and LinkedIn! Anyone can open it in a browser with no install required.
4
Update README with your live link
# Edit README.md — replace this line:
[🔗 Live Demo](https://your-username.github.io/ecommerce-dashboard)

# With your actual username:
[🔗 Live Demo](https://johndoe.github.io/ecommerce-dashboard)

# Then push the update:
git add README.md
git commit -m "Update README with live demo link"
git push
